# cuDNN 配置与 ldconfig 问题解决

我明白你在配置 cuDNN 时遇到了一些问题。`ldconfig` 命令报错的核心原因是 **库文件的符号链接不规范**，以及最后一个 **权限问题**。



***

## 1. 错误解释



* `libcudnn*.so.8 is not a symbolic link`

  这个警告是说，这些 `.so.8` 文件本应是**符号链接**（指向具体版本，如 `libcudnn.so``.8.9.1`），但现在它们是**实际的文件**或**无效链接**，`ldconfig` 无法正确更新缓存。

* `Can't link ... ``libnvoptix_loader.so``.1 to ``libnvoptix.so``.1`

  系统在尝试为 NVIDIA 相关库创建符号链接时失败，可能是因为文件不存在或权限不足。

* `Permission denied`

  你没有权限在 `/etc` 目录下创建临时缓存文件，所以 `ldconfig` 无法完成缓存更新。



***

## 2. 解决方案

### 步骤 1：修复 cuDNN 符号链接

首先进入你的 cuDNN 解压目录（你当前的目录），重新建立正确的符号链接。



```
cd \~/lmt/cudnn-linux-x86\_64-8.9.1.23\_cuda12-archive

\# 先删除旧的可能错误的链接（如果存在）

sudo rm -f /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\*.so.8

\# 重新创建符号链接（指向具体版本）

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_cnn\_train.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_cnn\_train.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_adv\_train.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_adv\_train.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_cnn\_infer.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_cnn\_infer.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_ops\_infer.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_ops\_infer.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_ops\_train.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_ops\_train.so.8

sudo ln -s /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_adv\_infer.so.8.9.1 /usr/local/cuda-12.1/targets/x86\_64-linux/lib/libcudnn\_adv\_infer.so.8
```

> 注意：请确保 
>
> `/usr/local/cuda-12.1/targets/x86_64-linux/lib/`
>
>  下有 
>
> `libcudnn*.so.8.9.1`
>
>  这些文件，如果版本号不同，需要对应修改。



***

### 步骤 2：修复 NVIDIA 驱动库链接（可选）

对于 `libnvoptix_loader.so``.1` 等错误，可以尝试重新安装 NVIDIA 驱动或手动创建链接：



```
\# 先检查是否存在 libnvoptix.so.1

ls /usr/lib/wsl/lib/libnvoptix.so.1

\# 如果存在，创建链接

sudo ln -sf /usr/lib/wsl/lib/libnvoptix.so.1 /usr/lib/wsl/lib/libnvoptix\_loader.so.1

\# 同样处理 libnvidia-gpucomp

ls /usr/lib/wsl/lib/libnvidia-gpucomp.so

sudo ln -sf /usr/lib/wsl/lib/libnvidia-gpucomp.so /usr/lib/wsl/lib/libnvidia-gpucomp.so.580.105.07
```

> 如果文件不存在，可能需要重新安装 NVIDIA 驱动 for WSL。



***

### 步骤 3：以 root 权限运行 ldconfig

最后，使用 `sudo` 权限更新缓存：



```
sudo ldconfig
```

如果还有权限错误，可以尝试：



```
sudo ldconfig -v
```

> `-v`
>
>  参数会显示详细过程，方便排查问题。



***

### 步骤 4：验证安装



```
\# 检查 cuDNN 版本

cat /usr/local/cuda-12.1/include/cudnn\_version.h | grep CUDNN\_MAJOR -A 2

\# 检查库文件是否被正确识别

ldconfig -p | grep cudnn
```

如果输出了 cuDNN 的版本号和库文件路径，说明安装成功。
